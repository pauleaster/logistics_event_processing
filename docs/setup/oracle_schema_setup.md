# Oracle Schema Setup

Setup steps for creating and validating the Oracle schema used by the logistics GPS event processing application.

## Schema scope

The first version of the application processes GPS crumb events only.

The Oracle schema contains three tables:

```text
drivers
vehicles
gps
```

The database stores successfully normalised GPS telemetry only.

Raw incoming messages and failed events are handled through structured Python logging and are not persisted to Oracle in this PoC.

## Data model

### `drivers`

Stores driver reference data.

```text
drivers
  driver_id      Oracle-generated integer primary key
  driver_code    external driver code from the GPS source system
  driver_name    driver display name
  active_flag    active status flag
  created_at     row creation timestamp
```

### `vehicles`

Stores vehicle reference data.

```text
vehicles
  vehicle_id            Oracle-generated integer primary key
  vehicle_code          external vehicle code from the GPS source system
  registration_number   vehicle registration number
  active_flag           active status flag
  created_at            row creation timestamp
```

### `gps`

Stores validated and normalised GPS telemetry.

```text
gps
  gps_id                  Oracle-generated integer primary key
  source_system           source system name
  external_event_id       external event identifier from the source system
  event_timestamp         timestamp from the GPS event
  driver_id               foreign key to drivers.driver_id
  vehicle_id              foreign key to vehicles.vehicle_id
  latitude                GPS latitude
  longitude               GPS longitude
  speed_kmh               vehicle speed in kilometres per hour
  heading_degrees         direction of travel in degrees
  gps_accuracy_m          reported GPS accuracy in metres
  battery_level_percent   driver device battery level percentage
  created_at              row creation timestamp
```

## Primary key strategy

All tables use Oracle identity columns for generated integer primary keys.

```sql
NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY
```

The primary key columns are:

```text
drivers.driver_id
vehicles.vehicle_id
gps.gps_id
```

The GPS event does not supply Oracle primary keys.

The incoming event supplies external source-system codes:

```text
driver_code
vehicle_code
```

The PL/SQL package resolves these codes to internal Oracle keys before inserting into `gps`.

```text
incoming driver_code  -> drivers.driver_code  -> drivers.driver_id
incoming vehicle_code -> vehicles.vehicle_code -> vehicles.vehicle_id
```

## GPS event contract

The first-pass GPS event payload is:

```json
{
  "source_system": "DRIVER_APP",
  "external_event_id": "gps-000001",
  "event_type": "GPS_CRUMB",
  "event_timestamp": "2026-06-24T09:15:30+10:00",
  "driver_code": "DRV1027",
  "vehicle_code": "VH-4412",
  "latitude": -37.813628,
  "longitude": 144.963058,
  "speed_kmh": 42.5,
  "heading_degrees": 87,
  "gps_accuracy_m": 8.5,
  "battery_level_percent": 76
}
```

GPS events are decoupled from shipment data. The first PoC does not include shipments or shipment event tables.

## Data mapping

```text
Incoming event field        Database target / behaviour
---------------------------------------------------------------------------
source_system               gps.source_system
external_event_id           gps.external_event_id
event_timestamp             gps.event_timestamp
driver_code                 lookup drivers.driver_code, store drivers.driver_id
vehicle_code                lookup vehicles.vehicle_code, store vehicles.vehicle_id
latitude                    gps.latitude
longitude                   gps.longitude
speed_kmh                   gps.speed_kmh
heading_degrees             gps.heading_degrees
gps_accuracy_m              gps.gps_accuracy_m
battery_level_percent       gps.battery_level_percent
```

## Validation approach

Python validates the incoming payload before calling Oracle.

Python validation rules:

```text
event_type must equal GPS_CRUMB
event_timestamp must be parseable
driver_code must be present
vehicle_code must be present
latitude must be between -90 and 90
longitude must be between -180 and 180
speed_kmh must be greater than or equal to 0
heading_degrees must be between 0 and 359
gps_accuracy_m must be greater than or equal to 0
battery_level_percent must be between 0 and 100
```

Oracle enforces relational and database-level rules.

Oracle validation rules:

```text
drivers.driver_code must be unique
vehicles.vehicle_code must be unique
vehicles.registration_number must be unique
gps.driver_id must reference drivers.driver_id
gps.vehicle_id must reference vehicles.vehicle_id
gps.external_event_id must be unique per source_system
gps.latitude must be between -90 and 90
gps.longitude must be between -180 and 180
gps.speed_kmh must be greater than or equal to 0 when supplied
gps.heading_degrees must be between 0 and 359 when supplied
gps.gps_accuracy_m must be greater than or equal to 0 when supplied
gps.battery_level_percent must be between 0 and 100 when supplied
```

Invalid events are logged by the Python application and are not inserted into Oracle.

## Create `oracle/schema.sql`

Create or replace the schema file:

```sql
-- oracle/schema.sql
--
-- Logistics Event Processing
-- GPS-first Oracle schema.
--
-- The database stores successfully normalised GPS telemetry only.
-- Raw and failed events are handled through structured Python logging.
SET SERVEROUTPUT ON;
PROMPT Creating logistics GPS schema...
--------------------------------------------------------------------------------
-- Drop existing tables
--------------------------------------------------------------------------------
BEGIN
  EXECUTE IMMEDIATE 'DROP TABLE gps PURGE';
EXCEPTION
  WHEN OTHERS THEN
    IF SQLCODE != -942 THEN
      RAISE;
    END IF;
END;
/
BEGIN
  EXECUTE IMMEDIATE 'DROP TABLE vehicles PURGE';
EXCEPTION
  WHEN OTHERS THEN
    IF SQLCODE != -942 THEN
      RAISE;
    END IF;
END;
/
BEGIN
  EXECUTE IMMEDIATE 'DROP TABLE drivers PURGE';
EXCEPTION
  WHEN OTHERS THEN
    IF SQLCODE != -942 THEN
      RAISE;
    END IF;
END;
/
--------------------------------------------------------------------------------
-- drivers
--------------------------------------------------------------------------------
CREATE TABLE drivers (
  driver_id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY
            CONSTRAINT pk_drivers PRIMARY KEY,
  driver_code VARCHAR2(50) NOT NULL,
  driver_name VARCHAR2(100) NOT NULL,
  active_flag CHAR(1) DEFAULT 'Y' NOT NULL,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  CONSTRAINT uq_drivers_driver_code
    UNIQUE (driver_code),
  CONSTRAINT ck_drivers_active_flag
    CHECK (active_flag IN ('Y', 'N'))
);
--------------------------------------------------------------------------------
-- vehicles
--------------------------------------------------------------------------------
CREATE TABLE vehicles (
  vehicle_id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY
             CONSTRAINT pk_vehicles PRIMARY KEY,
  vehicle_code VARCHAR2(50) NOT NULL,
  registration_number VARCHAR2(50) NOT NULL,
  active_flag CHAR(1) DEFAULT 'Y' NOT NULL,
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  CONSTRAINT uq_vehicles_vehicle_code
    UNIQUE (vehicle_code),
  CONSTRAINT uq_vehicles_registration_number
    UNIQUE (registration_number),
  CONSTRAINT ck_vehicles_active_flag
    CHECK (active_flag IN ('Y', 'N'))
);
--------------------------------------------------------------------------------
-- gps
--------------------------------------------------------------------------------
CREATE TABLE gps (
  gps_id NUMBER GENERATED BY DEFAULT ON NULL AS IDENTITY
         CONSTRAINT pk_gps PRIMARY KEY,
  source_system VARCHAR2(50) NOT NULL,
  external_event_id VARCHAR2(100) NOT NULL,
  event_timestamp TIMESTAMP NOT NULL,
  driver_id NUMBER NOT NULL,
  vehicle_id NUMBER NOT NULL,
  latitude NUMBER(9,6) NOT NULL,
  longitude NUMBER(9,6) NOT NULL,
  speed_kmh NUMBER(6,2),
  heading_degrees NUMBER(3),
  gps_accuracy_m NUMBER(8,2),
  battery_level_percent NUMBER(5,2),
  created_at TIMESTAMP DEFAULT SYSTIMESTAMP NOT NULL,
  CONSTRAINT fk_gps_driver
    FOREIGN KEY (driver_id)
    REFERENCES drivers (driver_id),
  CONSTRAINT fk_gps_vehicle
    FOREIGN KEY (vehicle_id)
    REFERENCES vehicles (vehicle_id),
  CONSTRAINT uq_gps_source_event
    UNIQUE (source_system, external_event_id),
  CONSTRAINT ck_gps_latitude
    CHECK (latitude BETWEEN -90 AND 90),
  CONSTRAINT ck_gps_longitude
    CHECK (longitude BETWEEN -180 AND 180),
  CONSTRAINT ck_gps_speed
    CHECK (speed_kmh IS NULL OR speed_kmh >= 0),
  CONSTRAINT ck_gps_heading
    CHECK (heading_degrees IS NULL OR heading_degrees BETWEEN 0 AND 359),
  CONSTRAINT ck_gps_accuracy
    CHECK (gps_accuracy_m IS NULL OR gps_accuracy_m >= 0),
  CONSTRAINT ck_gps_battery
    CHECK (
      battery_level_percent IS NULL
      OR battery_level_percent BETWEEN 0 AND 100
    )
);
--------------------------------------------------------------------------------
-- Indexes
--------------------------------------------------------------------------------
CREATE INDEX ix_gps_event_timestamp
  ON gps (event_timestamp);
CREATE INDEX ix_gps_driver_timestamp
  ON gps (driver_id, event_timestamp);
CREATE INDEX ix_gps_vehicle_timestamp
  ON gps (vehicle_id, event_timestamp);
CREATE INDEX ix_gps_source_system
  ON gps (source_system);
--------------------------------------------------------------------------------
-- Completion message
--------------------------------------------------------------------------------
BEGIN
  DBMS_OUTPUT.PUT_LINE('Schema created successfully.');
  DBMS_OUTPUT.PUT_LINE('Tables created: drivers, vehicles, gps');
END;
/
```

## Create `oracle/package.sql`

Create or replace the PL/SQL package file:

```sql
-- oracle/package.sql
--
-- Logistics Event Processing
-- GPS event processing package.
--
-- The Python application calls this package after validating and transforming
-- an incoming GPS event. The package resolves driver_code and vehicle_code
-- to internal Oracle primary keys, then inserts the normalised GPS row.
SET SERVEROUTPUT ON;
PROMPT Creating event_processing_pkg...
--------------------------------------------------------------------------------
-- Package specification
--------------------------------------------------------------------------------
CREATE OR REPLACE PACKAGE event_processing_pkg AS
  PROCEDURE insert_gps_crumb (
    p_source_system          IN VARCHAR2,
    p_external_event_id      IN VARCHAR2,
    p_event_timestamp        IN TIMESTAMP,
    p_driver_code            IN VARCHAR2,
    p_vehicle_code           IN VARCHAR2,
    p_latitude               IN NUMBER,
    p_longitude              IN NUMBER,
    p_speed_kmh              IN NUMBER,
    p_heading_degrees        IN NUMBER,
    p_gps_accuracy_m         IN NUMBER,
    p_battery_level_percent  IN NUMBER
  );
END event_processing_pkg;
/
SHOW ERRORS PACKAGE event_processing_pkg;
--------------------------------------------------------------------------------
-- Package body
--------------------------------------------------------------------------------
CREATE OR REPLACE PACKAGE BODY event_processing_pkg AS
  PROCEDURE insert_gps_crumb (
    p_source_system          IN VARCHAR2,
    p_external_event_id      IN VARCHAR2,
    p_event_timestamp        IN TIMESTAMP,
    p_driver_code            IN VARCHAR2,
    p_vehicle_code           IN VARCHAR2,
    p_latitude               IN NUMBER,
    p_longitude              IN NUMBER,
    p_speed_kmh              IN NUMBER,
    p_heading_degrees        IN NUMBER,
    p_gps_accuracy_m         IN NUMBER,
    p_battery_level_percent  IN NUMBER
  ) AS
    v_driver_id   drivers.driver_id%TYPE;
    v_vehicle_id  vehicles.vehicle_id%TYPE;
  BEGIN
    --------------------------------------------------------------------------
    -- Resolve active driver
    --------------------------------------------------------------------------
    BEGIN
      SELECT driver_id
      INTO v_driver_id
      FROM drivers
      WHERE driver_code = p_driver_code
        AND active_flag = 'Y';
    EXCEPTION
      WHEN NO_DATA_FOUND THEN
        RAISE_APPLICATION_ERROR(
          -20001,
          'Active driver not found for driver_code: ' || p_driver_code
        );
      WHEN TOO_MANY_ROWS THEN
        RAISE_APPLICATION_ERROR(
          -20002,
          'Multiple active drivers found for driver_code: ' || p_driver_code
        );
    END;
    --------------------------------------------------------------------------
    -- Resolve active vehicle
    --------------------------------------------------------------------------
    BEGIN
      SELECT vehicle_id
      INTO v_vehicle_id
      FROM vehicles
      WHERE vehicle_code = p_vehicle_code
        AND active_flag = 'Y';
    EXCEPTION
      WHEN NO_DATA_FOUND THEN
        RAISE_APPLICATION_ERROR(
          -20003,
          'Active vehicle not found for vehicle_code: ' || p_vehicle_code
        );
      WHEN TOO_MANY_ROWS THEN
        RAISE_APPLICATION_ERROR(
          -20004,
          'Multiple active vehicles found for vehicle_code: ' || p_vehicle_code
        );
    END;
    --------------------------------------------------------------------------
    -- Insert normalised GPS row
    --------------------------------------------------------------------------
    INSERT INTO gps (
      source_system,
      external_event_id,
      event_timestamp,
      driver_id,
      vehicle_id,
      latitude,
      longitude,
      speed_kmh,
      heading_degrees,
      gps_accuracy_m,
      battery_level_percent
    )
    VALUES (
      p_source_system,
      p_external_event_id,
      p_event_timestamp,
      v_driver_id,
      v_vehicle_id,
      p_latitude,
      p_longitude,
      p_speed_kmh,
      p_heading_degrees,
      p_gps_accuracy_m,
      p_battery_level_percent
    );
  EXCEPTION
    WHEN DUP_VAL_ON_INDEX THEN
      RAISE_APPLICATION_ERROR(
        -20005,
        'Duplicate GPS event for source_system/external_event_id: '
        || p_source_system || '/' || p_external_event_id
      );
    WHEN OTHERS THEN
      RAISE;
  END insert_gps_crumb;
END event_processing_pkg;
/
SHOW ERRORS PACKAGE BODY event_processing_pkg;
--------------------------------------------------------------------------------
-- Completion message
--------------------------------------------------------------------------------
BEGIN
  DBMS_OUTPUT.PUT_LINE('Package created successfully.');
  DBMS_OUTPUT.PUT_LINE('Package: event_processing_pkg');
  DBMS_OUTPUT.PUT_LINE('Procedure: insert_gps_crumb');
END;
/
```

The package exposes one first-pass procedure:

```text
event_processing_pkg.insert_gps_crumb
```

The procedure accepts `driver_code` and `vehicle_code` from the validated GPS event, resolves them to `drivers.driver_id` and `vehicles.vehicle_id`, and inserts the normalised row into `gps`.

Errors raised by this package are handled by the Python application and written to structured logs.

```sql
SELECT
  g.gps_id,
  g.source_system,
  g.external_event_id,
  g.event_timestamp,
  g.driver_id,
  d.driver_code,
  g.vehicle_id,
  v.vehicle_code,
  g.latitude,
  g.longitude,
  g.speed_kmh,
  g.heading_degrees,
  g.gps_accuracy_m,
  g.battery_level_percent,

JOIN drivers d
  ON d.driver_id = g.driver_id
JOIN vehicles v
  ON v.vehicle_id = g.vehicle_id
```
